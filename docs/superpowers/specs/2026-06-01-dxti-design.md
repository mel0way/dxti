# DXTI（大学TI）设计文档

> 日期：2026-06-01  
> 状态：已确认

## 一、项目概述

DXTI（大学TI）是一个类似 MBTI 的大学匹配测试应用。用户通过 40 道 BuzzFeed 风格的情景选择题，测出自己最匹配的 985/211 大学，生成可分享的结果卡片。另设"严谨测试"入口，伪装成严肃测评，实际跳转至高考历年真题搜索页。

**目标用户**：高中生、大学生  
**变现模式**：流量增长后接入广告（Google AdSense / 微信广告）  
**传播引擎**：结果页分享卡片 → 朋友圈/群聊/微博 → 病毒传播

## 二、技术栈

- **框架**：Uni-app (Vue 3)
- **构建**：Vite (H5 模式)
- **部署**：H5 网页优先（Vercel/Netlify），后续可发微信小程序
- **依赖**：html2canvas（分享图生成）

## 三、测评体系

### 3.1 四个维度

| 维度 | 方向 | 描述 |
|------|------|------|
| 学术型 ↔ 实践型 | A ↔ P | 图书馆战神 vs 动手实干派 |
| 综合型 ↔ 专精型 | C ↔ S | 什么涉猎 vs 深耕一个领域 |
| 都市型 ↔ 校园型 | U ↔ M | 城市霓虹 vs 校园宁静 |
| 严谨型 ↔ 自由型 | R ↔ F | 计划表狂魔 vs 随性自由派 |

### 3.2 题目设计

- **风格**：BuzzFeed 情景选择，如"你的手机相册里最多的是什么？"
- **数量**：40 题，每个维度 10 题，维度题随机穿插
- **选项**：每题 4 个选项，每个选项为某个维度方向 +1 分
- **各维度满分**：10 分

## 四、匹配算法

### 4.1 评分

每题选择后，对应维度方向 +1 分。40 题完成后，用户在 4 个维度各得一个 0-10 的方向分，构成用户向量 `[学术分, 综合分, 都市分, 严谨分]`。

### 4.2 大学画像

每所大学预设 4 维向量（1-10 分制），例如：

```json
{
  "name": "清华大学",
  "vector": [9, 6, 7, 9]
}
```

50 所大学均预设画像数据。

### 4.3 匹配度计算

使用**余弦相似度**计算用户向量与大学向量的匹配度，归一化为百分比。按匹配度降序排列，取前 3 名展示。

### 4.4 人格类型判定

根据用户 4 维得分的高低，取每个维度的主导方向（如学术分 > 实践分则为学术型），组合成 16 种人格之一。

## 五、页面结构

### 5.1 首页

- 顶部：DXTI Logo + "大学人格类型测试"
- 主标题："哪所大学是你的命中注定？"
- 两个纵向卡片按钮：
  - **模拟测试**："40道有趣情景题，找出你的大学人格"
  - **严谨测试**："最准确最真实的测试" → 点进去是高考真题
- 底部：XX 人已找到命中大学（社交证明）

### 5.2 测试页

- 顶部进度条 + "第 X / 40 题"
- 题目文字 + 4 个选项卡片
- 选中选项后高亮，点击"下一题"进入下一题
- 支持返回上一题修改
- 最后一题为"查看结果"

### 5.3 结果页

- 人格类型名称（如"博雅通才"） + 描述
- 匹配度 Top 3 大学（金/银/铜卡片）
- "生成分享图"按钮：渲染为图片（含类型名、Top 3 大学、二维码）
- "重新测试"链接

### 5.4 严谨测试页（真题搜索）

- 标题："严谨测试" + 副标题"最准确最真实的测试 —— 历年高考真题"
- 搜索框
- 筛选标签：省份（全国卷/北京卷/新高考Ⅰ卷...）、年份（2025/2024/2023...）、科目（语/数/英/物/化/生/政/史/地）
- 链接列表：每条显示"年份+试卷+科目"，点击跳转至外部真题网站

## 六、分享传播机制

1. 结果页点击"生成分享图"
2. 前端将结果卡片渲染为图片（html2canvas）
3. 用户保存图片/直接分享到微信/朋友圈/微博
4. 图片包含：人格类型名 + 匹配 Top 3 大学 + 匹配度 + 二维码
5. 朋友扫码或搜索进入 DXTI → 循环传播

- **配色**：紫 (#88619a) · 青 (#33b5a9) · 金 (#f4c542) · 粉 (#e2556d)
